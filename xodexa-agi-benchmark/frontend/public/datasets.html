Xodexa is built so that memorizing the public set cannot earn an official score. The corpus is split into six layers of increasing contamination resistance: answer keys for official tasks live server-side only, dynamic generators mint fresh variants on every run, and every release is pinned by a SHA-256 checksum and signed by the central authority.
The six-layer corpus

Why this resists contamination

  • Public ≠ official. Layer 1's 1,000 validation tasks are open with public keys for self-testing, but they never produce an official score.
  • Hidden official set. Layer 2's 500 tasks are private; answer keys sit in git-ignored server_keys and are applied only by the central scoring authority.
  • Dynamic generators. Layer 3 mints fresh task variants per run, so a leaked snapshot is stale the next time you submit.
  • Checksum + signature. Every release pins a SHA-256 and is signed (Ed25519) so the exact tasks scored can be proven after the fact.
  • Rotation. The private set rotates; a model must hold its readiness level across rotations, not just one snapshot.
